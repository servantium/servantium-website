## General

### What is Servantium?

Servantium is a CPQ (Configure, Price, Quote) and engagement management platform built for professional services businesses. It helps you scope engagements, create accurate pricing, generate proposals, and manage delivery — with institutional memory built in.

### How is Servantium different from PSA software?

Traditional PSA tools bolt project management onto sales software. Servantium takes a different approach — it's purpose-built for services, with an integrated system where sales, delivery, and learning share context. The engagement lifecycle is the core abstraction, not timesheets.

### What types of businesses use Servantium?

Consulting firms, implementation teams, managed services providers, creative agencies, and technical services firms. If you deliver repeatable (but customizable) engagements and want to stop reinventing every project from scratch, Servantium is for you.

## Features

### How does the AI work?

AI in Servantium learns from your encoded patterns — past engagements, pricing decisions, delivery outcomes. It assists with entity extraction from notes, similar engagement matching, and estimate accuracy improvement. Human judgment stays in control at every step.

### What integrations does Servantium support?

Servantium integrates bi-directionally with **Salesforce** — opportunities, contacts, and engagement data flow both ways. We also have a REST API and CSV import for migrating existing data. More integrations are in development based on customer needs.

### Can I import existing data?

Yes. CSV import is available for contacts, accounts, and engagement history. We also offer guided data migration for Teams and Enterprise customers.

## Security

### Is my data secure?

Yes. All data is encrypted at rest and in transit. Servantium is built on Google Cloud Platform (Firebase) with enterprise-grade security controls. Each organization's data is isolated at the application level.

### Where is my data stored?

Data is stored in Google Cloud's US Central region. Enterprise customers can discuss data residency requirements.
