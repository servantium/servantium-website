The Settings screen is the administration hub for your organization. Access it from the **gear icon** in the sidebar. Settings controls templates, the Service Catalog, discounts, tags, and data import.

## Organization info

The **Info** tab contains your organization's basic details:

| Field | Description |
|-------|-------------|
| **Name** | Organization display name (required) |
| **Resource Planning Interval** | Default interval for resource plans: Day, Week, or Month |
| **Description** | A brief description of the organization |
| **Address** | Physical address |
| **Phone** | Organization phone number |
| **Email** | Organization email address |
| **Website** | Organization website URL |

<Aside type="tip">
The **Resource Planning Interval** setting determines the default granularity for new resource plans across the organization. Individual plans can still override this default.
</Aside>

## Engagement templates

The **Engagement Templates** tab manages the templates that define engagement types. See the [Engagements guide](/help/guides/engagements) for how templates work.

### Managing engagement templates

<Steps>

1. Navigate to **Settings > Engagement Templates**.

2. The list shows all templates for your organization.

3. Click a template to edit its name and fields.

4. Click **Create** to add a new template.

</Steps>

Each template has:

- **Name** -- The template label shown in the type selector
- **Fields** -- A configurable list of custom fields (text, chip, slider, nested template, image)
- **Slideshow** -- Toggle for presentation-style display of template fields

Templates can also be generated or modified using AI by providing natural language instructions.

## Quote templates

The **Quote Templates** tab manages reusable quote structures.

Quote templates encode:

- Section names and ordering
- Default line items (roles, materials)
- Default costs and markup percentages
- Dynamic formulas that evaluate against engagement data

### Creating a quote template

<Steps>

1. Navigate to **Settings > Quote Templates**.

2. Click **Create** to add a new template.

3. Define sections and line items. Alternatively, save an existing quote as a template directly from the quote editor.

</Steps>

When a quote template is applied to a new quote, a backend trigger hydrates the template -- evaluating any formulas and populating the quote with the template's structure.

## Document templates

The **Document Templates** tab manages the DOCX templates used for document generation. See the [Documents guide](/help/guides/documents) for details on how templates are used.

<Steps>

1. Navigate to **Settings > Document Templates**.

2. Click **Create** to add a new template.

3. Enter a **title** and **upload** the template DOCX file.

</Steps>

Document templates use merge fields that reference engagement, quote, organization, and profile data during generation.

## Service Catalog (items)

The **Items** tab shows your organization's master catalog of resources and materials. Each item has:

| Field | Description |
|-------|-------------|
| **Name** | Item name (e.g., "Senior Developer," "Cloud License") |
| **Type** | Resource or Material |
| **Cost** | Standard per-unit cost |
| **Markup** | Default markup percentage |
| **Status** | Active or inactive |

The Service Catalog is a **learning system**. It updates automatically as items are added or modified in quotes across your organization. You can also manage items manually from this screen.

<Aside type="note">
The Service Catalog powers the autocomplete when adding line items to quotes. Keeping it up to date ensures consistent pricing across your team.
</Aside>

## Discounts

The **Discounts** tab manages quantity-based pricing tiers that are automatically applied during quote calculation.

Each discount rule has:

| Field | Description |
|-------|-------------|
| **Min** | Minimum quantity for the discount to apply |
| **Max** | Maximum quantity threshold |
| **Discount** | Discount percentage or amount |
| **Item** | Optionally restrict to a specific catalog item |

When a quote line item's quantity falls within a discount range, the backend applies the discount automatically during total recalculation.

## Tags

The **Tags** tab manages the tag categories and individual tags used throughout Servantium.

### Tag categories

Tags are organized into **categories**, each with:

- **Category name** -- The grouping label (e.g., "Industry," "Priority," "Service Line")
- **Color** -- A hex color for visual identification (defaults to Servantium green)
- **Tags** -- The individual tag values within the category

### Managing tags

<Steps>

1. Navigate to **Settings > Tags**.

2. Create a **tag category** by clicking **Create** and entering a name and color.

3. Within a category, add individual tags by name.

4. Tags are available organization-wide on engagements, resources, accounts, and other objects.

</Steps>

<Aside type="tip">
Design your tag categories around how your organization naturally groups work. Common categories: Industry, Service Line, Technology, Region, Priority.
</Aside>

## Data import

The **Dataload** tab provides bulk data import functionality for migrating data into Servantium.

### Creating a data import

<Steps>

1. Navigate to **Settings > Dataload**.

2. Click **Create** to set up a new import.

3. Configure the import:
   - **Name** -- A descriptive label for this import job
   - **Object** -- The target Firestore collection (e.g., "engagements," "accounts," "resources")
   - **Mapping** -- A field mapping definition that maps CSV columns to Firestore fields
   - **File** -- Upload the CSV file to import

4. Click **Create** to save the import configuration.

5. Toggle the **load** flag to trigger the import process.

</Steps>

<Aside type="caution">
Data imports write directly to your organization's data. Test with a small file first to verify your mapping is correct before importing large datasets.
</Aside>

## Profile settings

Your personal profile is accessible from the **Profile menu** in the top bar. Profile settings include:

- **Email** -- Your login email (read-only, set during registration)
- **Organizations** -- List of organizations you belong to
- **Selected organization** -- The currently active organization
- **Display name** -- Change your name via the name change dialog

### Switching organizations

If you belong to multiple organizations, select a different one from the profile menu. All screens and data update to reflect the selected organization.
