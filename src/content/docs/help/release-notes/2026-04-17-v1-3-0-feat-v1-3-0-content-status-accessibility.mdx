v1.3.0 adds content status controls, WCAG 2.1 AA accessibility fixes, and asteroids game improvements.

## Content status field

All content collections now support a `status` field with three values:

- `published` (default)  -  visible in production and development
- `draft`  -  visible in development only; hidden in production
- `hidden`  -  hidden in all environments

The status field applies to:

- Docs (`/help/*`)
- Blog posts (`/blog/*`)
- Customer stories (`/resources/customers/*`)
- Press mentions (`/resources/press/*`)
- Resources (`/resources/*`)
- Comparisons (`/compare/*`)

The new `filterByStatus` helper in `src/lib/content-filter.ts` enforces these rules across all collection queries.

### Migration

Existing content defaults to `published`. No action required unless you want to mark items as draft or hidden.

To mark content as draft:

```yaml
---
title: "Work in progress"
status: hidden
---
```

To hide content permanently:

```yaml
---
title: "Deprecated page"
status: "hidden"
---
```

## Accessibility improvements

This release brings the site to WCAG 2.1 AA compliance:

- **ARIA labels**  -  Cal.com iframe, cookie consent dialog, FAQ accordions, contact form status messages
- **Focus traps**  -  Mobile menu, search modal, contact form success overlay all trap focus when open
- **Keyboard navigation**  -  Escape key closes cookie consent, mobile menu, and search modal; P key pauses asteroids game
- **Semantic HTML**  -  Footer headings now use `role="heading"` with `aria-level="3"`; FAQ answers use `role="region"` with `aria-labelledby`
- **Motion preferences**  -  Section fade-in animations respect `prefers-reduced-motion: reduce`

Focus is returned to the trigger element when modals close.

## Asteroids game enhancements

The easter egg asteroids game (`/js/asteroids.js`) now includes:

- **Pause**  -  Press P to pause; escape closes the game
- **High scores**  -  Persisted to localStorage after first play (opt-in)
- **Thrust particles**  -  Visual feedback when ship accelerates
- **Level indicators**  -  "Level N" overlay when advancing
- **Game-over screen**  -  Witty messages, score display, high score tracking
- **Double-tap hint**  -  "Double-tap to play" appears below the trigger
- **Window resize handling**  -  Game dimensions update on viewport resize

High scores only use localStorage after the player has completed one game, avoiding opt-in consent issues.
